# CHAPTER 02 — CONTROL FLOW

This chapter covers Python's control flow mechanisms, including conditional statements, loops, and comprehensions. These fundamental structures enable you to control the execution flow of your programs and process data efficiently.

## Files Overview

### `0-if_mechanics.py`
**Conditional Logic and Truth Values**
- Falsy values and truthiness rules in Python
- Short-circuit evaluation with `and`/`or` operators
- Ternary conditional expressions for concise decisions
- Walrus operator (`:=`) for assignment in expressions
- Identity vs equality comparison (`is` vs `==`)
- Integer caching and object identity nuances

### `1-loops_complete.py`
**Iteration and Loop Control**
- **for loops**: Iterating over any iterable, range usage patterns
- **enumerate()**: Getting index and value together
- **zip()**: Iterating multiple sequences simultaneously
- **for...else clause**: Executing code when loops complete without break
- **while loops**: Including walrus operator patterns
- **Loop control**: `continue`, `break`, and `pass` statements
- Best practices for iteration in Python

### `2-comprehensions_complete.py`
**Comprehensions and Generator Expressions**
- **List comprehensions**: Creating lists concisely with filtering and mapping
- **Dict comprehensions**: Building dictionaries from iterables
- **Set comprehensions**: Creating unique value sets
- **Generator expressions**: Lazy evaluation for memory efficiency
- **Nested comprehensions**: Handling multi-dimensional data
- **Performance considerations**: When to use each type
- Memory usage patterns and optimization strategies

## Learning Path

1. Start with `0-if_mechanics.py` to master conditional logic and Python's truth model
2. Study `1-loops_complete.py` to understand iteration patterns and loop control
3. Complete with `2-comprehensions_complete.py` to write concise, efficient data transformations

## Key Concepts Covered

- **Truthiness**: Understanding how Python evaluates objects in boolean contexts
- **Short-circuiting**: Leveraging `and`/`or` for efficient conditional logic
- **Lazy evaluation**: Using generators and range objects for memory efficiency
- **Functional patterns**: Map, filter, and reduce through comprehensions
- **Pythonic iteration**: Moving away from index-based loops to iterator-based patterns

Each file demonstrates idiomatic Python patterns with practical examples, performance considerations, and common pitfalls to avoid. The examples progress from basic concepts to advanced patterns, providing both learning material and reference code.
